# 表格识别功能说明

## 概述

Wrong Notebook 支持识别和处理题目图片中的表格内容。AI 会自动将表格转换为 Markdown 格式，确保表格数据完整保留。

## 支持的表格类型

### 1. 标准表格
简单的行列结构表格，AI 会转换为标准 Markdown 表格语法：

```markdown
| 列标题1 | 列标题2 | 列标题3 |
|---------|---------|---------|
| 数据1   | 数据2   | 数据3   |
| 数据4   | 数据5   | 数据6   |
```

**适用场景**：
- 数学统计表
- 物理实验数据表
- 化学元素对照表
- 生物分类表

### 2. 复杂表格
包含合并单元格、多级表头或不规则布局的表格。

**处理方式**：
- 优先尝试用 Markdown 表格近似表示
- 如果无法准确表达，会在表格前添加文字说明结构
- 使用简化的 Markdown 表格 + 注释

**示例**：
```markdown
> 注：第1行为主标题，横跨3列；第2-3行为数据行

| 项目 | 数值A | 数值B |
|------|-------|-------|
| 测试1 | 10 | 20 |
| 测试2 | 15 | 25 |
```

### 3. 特殊情况

#### 手写表格
- AI 会尽力识别手写内容
- 不确定的内容用 `[?]` 标注
- 建议使用清晰的打印体表格以获得最佳识别效果

#### 模糊表格
- 如果表格不清晰，AI 会在表格前注明"（表格内容可能不完整）"
- 建议重新拍摄清晰的图片

#### 图表混合
- 如果表格旁边有图形，AI 会用文字说明位置关系
- 确保表格和图形的关联关系被正确识别

## 表格完整性保证

AI 在处理表格时会确保：

1. ✅ **所有单元格内容**：包括空单元格（用 `-` 或空格表示）
2. ✅ **表格标题和编号**：如"表1"、"表2"等
3. ✅ **单位和注释**：保留表格中的单位标注和说明文字
4. ✅ **数据对齐关系**：保持数据的分组和对应关系
5. ✅ **数学公式**：表格中的公式使用 LaTeX 语法（如 `$\frac{1}{2}$`）
6. ✅ **上下文信息**：保留表格前后的说明文字

## 使用建议

### 拍摄技巧
1. **清晰度**：确保表格边框和文字清晰可见
2. **光线**：避免反光和阴影
3. **角度**：尽量垂直拍摄，避免倾斜
4. **完整性**：确保表格的所有行列都在画面内

### 最佳实践
1. **单独拍摄**：如果表格很大，可以单独拍摄表格部分
2. **标注清晰**：确保表格标题、列标题清晰
3. **避免遮挡**：不要让手指或其他物体遮挡表格内容
4. **检查结果**：识别后检查表格数据是否完整准确

## 技术实现

### AI Prompt 设计
系统使用专门的表格处理指令，指导 AI：
- 识别表格结构
- 提取所有单元格数据
- 转换为 Markdown 格式
- 处理特殊情况（合并单元格、多级表头等）

### Markdown 渲染
前端使用 `remark-gfm` 插件支持 GitHub Flavored Markdown 表格语法，确保表格正确显示。

## 常见问题

### Q: 表格识别不准确怎么办？
**A**: 
1. 检查图片清晰度，重新拍摄
2. 确保表格完整在画面内
3. 手动编辑题目文本，修正表格内容

### Q: 复杂表格显示不正确？
**A**: 
- Markdown 表格语法有限制，无法完美表示所有复杂表格
- AI 会尽量用文字说明 + 简化表格的方式处理
- 可以在编辑时手动调整表格格式

### Q: 表格中的公式识别错误？
**A**: 
- 确保公式清晰可见
- 可以手动编辑，使用 LaTeX 语法修正公式
- 行内公式：`$公式$`
- 块级公式：`$$公式$$`

### Q: 支持哪些表格格式？
**A**: 
- 支持所有可以用 Markdown 表格表示的格式
- 不支持：单元格背景色、边框样式、单元格内换行
- 建议：使用简单的行列结构表格
